PROBLEM_SPACE

Stateless AI Is Broken AI

Telegram hosts over 500 million monthly active mini-app users — yet every AI-powered bot forgets you the instant you close the chat. Developers are forced to bolt on centralized Postgres instances, Redis caches, and proprietary auth systems, destroying the trustless promise of Web3-native applications. The result: users hand their behavioral data to whoever deployed the bot, with zero visibility and zero recourse.

The problem is architectural. TON mini-apps run in ephemeral WebView contexts. There is no native persistence primitive for AI agent state. Context windows reset. Preferences vanish. Conversation history evaporates. Every interaction starts from zero — making sophisticated, personalized AI agents technically impossible to build at scale on Telegram without compromising on decentralization.

And the economic reality compounds it: running a centralized memory backend for 500M users costs millions per year in infrastructure. Developers either underbuild their AI features or introduce the exact honeypot attack surface that blockchain users came to avoid. The memory layer is the missing primitive in the entire TON AI stack.

SOLUTION_ARCHITECTURE

Wallet-Keyed Memory, Anchored On-Chain

Syncpad introduces a three-layer architecture that makes persistent, sovereign AI memory economically viable at Telegram scale. Client-side encryption means the server never sees plaintext. TON Storage provides decentralized blob persistence. A FunC smart contract on TON mainnet anchors every write as an auditable on-chain event — creating a tamper-proof ledger of every agent interaction.

ENCRYPT_LAYERMemory is encrypted with AES-256-GCM before it ever leaves the browser, keyed directly to the user's TON wallet address. Zero-knowledge to Syncpad infrastructure.
STORE_LAYEREncrypted blobs are committed to TON Storage as versioned, TL-B serialized cells. Content-addressed, immutable, and retrievable by any authorized app.
ANCHOR_LAYERA FunC smart contract writes content hashes to TON mainnet, emitting auditable events on every write. The on-chain record is the source of truth.
SDK_LAYERThree functions: write(), read(), diff(). Publish to npm, import in any Telegram mini-app, authenticate with TON Connect 2.0. Ship persistent AI in one afternoon.

EXECUTION_FLOW

How a Memory Write Executes

01

WALLET_AUTHENTICATION

User opens a Telegram mini-app and authenticates with TON Connect 2.0. Their wallet public key becomes the root of their memory namespace — no username, no email, no account. The key is the identity.

02

CLIENT_SIDE_ENCRYPT_AND_SERIALIZE

The TypeScript SDK encrypts the agent context payload with AES-256-GCM on-device, serializes it into a TL-B cell structure, and prepares a versioned memory shard with an optional TTL. The encrypted blob is pushed to TON Storage.

03

ON_CHAIN_HASH_ANCHOR

The SDK submits a lightweight transaction to the Syncpad FunC smart contract on TON mainnet. The contract records the content hash, app identifier, timestamp, and TTL — emitting an auditable write event visible to the wallet owner.

04

READ_DIFF_AND_RESUME

On next session, read() fetches and decrypts the latest shard. diff() surfaces what changed between sessions, enabling AI agents to intelligently resume context without replaying entire histories. Revoke any app's access instantly via a signed TON transaction.
